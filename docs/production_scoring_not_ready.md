# Production Scoring Is Not Ready

Round 18 improves the map and research loop, but it does not justify production
score changes.

The reason is simple:

> A mapped theme is not the same as validated evidence.

Example:

- `스테이블코인` is now mapped.
- But Green would require regulated issuance, real transaction volume, fee economics, and low regulatory/security risk.
- A policy headline alone is only a research route.

## What Is Ready

- Raw theme list is captured.
- Theme map v0.5 exists.
- Unmatched audit exists.
- Case library v03 exists.
- Shadow score profiles exist.
- Shadow normalizer exists.

## What Is Not Ready

- Many cases still need price backfill.
- Several archetypes still lack enough positive and counterexample cases.
- Score profiles have not passed score-price alignment.
- Red/event/speculative themes still need strong guardrails.

## Hard Rules

- Do not lower Stage 3-Green thresholds.
- Do not use theme names as score evidence.
- Do not use case records as production candidate-generation input.
- Do not fabricate dates, prices, contract fields, or financial fields.
