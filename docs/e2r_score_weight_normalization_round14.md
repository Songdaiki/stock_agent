# Round 14 Score-Weight v0.4

Round 14 converts the long theme list into a calibration matrix.

The important rule is simple:

```text
theme tag -> large sector -> E2R archetype -> evidence field -> score-weight hypothesis -> price-path validation
```

Example:

- `스테이블코인` is a search/routing tag.
- It does not create score by itself.
- It only becomes stronger when regulation approval, transaction volume, fee economics, and recurring revenue are visible.

## What Changed

- Added a report-only Round 14 v0.4 score-weight matrix.
- Added theme tag rows for convenience retail, insurance, fintech, tokenization, K-beauty OEM/ODM, ESS/recycling, renewables, auto components, CDMO, platform/security, robotics, construction/PF, disease events, speculative science, and agri/livestock cycles.
- Added Green / Watch / RedTeam posture output.
- Added planned success and counterexample case rows.

## What Did Not Change

- Production scoring did not change.
- StageClassifier thresholds did not change.
- Theme tags are not score inputs.
- Case labels are not candidate-generation inputs.

## Operator Interpretation

Green-possible groups still need durable evidence:

- EPS/FCF or OP improvement
- structural visibility
- valuation rerating room
- low RedTeam risk
- price-path alignment

Watch/Yellow groups can be useful Layer-1 or Stage-2 candidates, but need more evidence before Stage 3-Green.

RedTeam-first groups are mainly used to block false positives. For example, `초전도체`, `엠폭스`, or `코인 테마` can move prices, but without recurring revenue and EPS/FCF evidence they should not be treated as structural E2R.
