# Round 16 Theme Coverage v0.5

Round 16 consolidates the previous theme-taxonomy work into a full coverage map.

The key rule is still:

```text
Raw Theme Tag
-> large sector
-> E2R archetype / sub-archetype
-> Green policy
-> must-have evidence
-> red-flag evidence
-> score-weight group
-> case library
-> price-path validation
```

## Easy Examples

`초전도체` is not a score input.

It routes to speculative-science RedTeam checks. Unless commercialization, revenue conversion, and repeat EPS/FCF evidence appear, it should not become Stage 3-Green.

`스테이블코인` is also not a score input.

It routes to digital-finance research. It needs regulation approval, actual issuance/volume, fee economics, and recurring revenue before it can move beyond theme status.

`전력설비` can be Green-possible, but only with contract quality, backlog, CAPA/lead-time evidence, margin path, and EPS/FCF revision.

## What Changed

- Added a Round 16 coverage module that merges Round 10, 12, 14, and 15 theme mappings.
- Added a 12-sector / 70+ sub-archetype / 200+ unique-theme coverage report.
- Added score-weight group hypotheses for Green, Watch/Yellow, and RedTeam-first buckets.
- Added CSV outputs for theme routing and score group review.

## What Did Not Change

- Production scoring did not change.
- StageClassifier thresholds did not change.
- Theme tags are not scoring evidence.
- Case/theme labels are not candidate-generation inputs.
