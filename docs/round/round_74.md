좋아. **R7 Loop 3까지 끝났으니, 이번은 R8 Loop 3 — 플랫폼·콘텐츠·SW·보안**으로 넘어간다.

R8은 게임, 엔터, 미디어 콘텐츠, 광고, 클라우드, AI SW, 컨택센터, IT보안, 딥페이크, 생체인식, CCTV, 메타버스, NFT를 흡수하는 대섹터다. Theme Tag Map 기준으로 클라우드/SaaS와 보안은 Watch-to-Green 가능성이 있지만, 게임·엔터·미디어는 IP·광고 cycle·안전규제를 봐야 하고, NFT·메타버스는 실적 전까지 Red/Watch로 둬야 한다.

서생원식 기준으로 R8의 핵심은 “AI 기능이 있다”가 아니라 **반복매출·ARR·고객 lock-in·OPM/FCF 개선·가격경로 리레이팅이 실제로 같이 일어나는가**다. 즉 AI demo, 유저 수, 신작 기대, 보안 테마가 아니라 숫자로 확인되는 workflow 고착과 이익 체급 변화가 필요하다.

2026년 1분기에는 매출이 전년 대비 85% 증가한 16.3억 달러였고, 미국 commercial revenue는 133%, 미국 government revenue는 84% 증가했다. 회사는 AI workflow가 “governed, attributed and auditable”하다는 점을 강조했지만, 실적 후 주가는 시간외에서 2.7% 하락했다. 즉 구조적 성공이면서도 valuation/crowding을 감시해야 하는 4B 사례다. ([마켓워치][3])

# 10. 가격경로 검증계획

## R8 Loop 3 공통 검증 방식

```text
1. 각 case의 stage1_date, stage2_date, stage3_date를 지정한다.
2. 해당 날짜의 종가를 stage_price로 저장한다.
3. MFE_30D / 90D / 180D / 1Y / 2Y를 계산한다.
4. MAE_30D / 90D / 180D / 1Y를 계산한다.
5. peak_price와 drawdown_after_peak를 계산한다.
6. ARR, billings, bookings, ad revenue, churn, OPM, FCF, 보안·법적 이벤트와 가격경로를 비교한다.
```

## Loop 3에서 새로 강제할 판정

```text
B2B_SAAS_STRUCTURAL_SUCCESS:
B2B 반복매출, 낮은 churn, OPM 개선이 확인됨.

AI_SOFTWARE_REVENUE_ALIGNED:
AI 기능이 실제 계약·매출·FCF로 연결.

AI_SOFTWARE_4B_VALUATION:
실적은 강하지만 valuation/crowding 때문에 주가가 둔화.

OBSERVABILITY_AI_ALIGNED:
AI workload monitoring이 ARR·guidance·가격경로와 동행.

STREAMING_AD_PLATFORM_ALIGNED:
광고 tier user growth + ad revenue + own ad-tech가 확인됨.

ADTECH_PREMIUM_MISS:
premium valuation 상태에서 revenue/guidance miss로 큰 drawdown.

SECURITY_TRUST_BREAK:
대형 보안 장애, 고객피해, 소송, 갱신위험.
```

# R8 Loop 3 결론

이번 3회차에서 R8은 더 좁혀졌다.

```text
Green 가능:
B2B SaaS / ERP 중 ARR·churn·FCF가 확인된 기업
AI software 중 계약·workflow·auditable AI·FCF가 확인된 기업
Observability / AIOps 중 AI workload monitoring이 ARR로 연결된 기업
Security 중 billings·ARR·renewal이 좋고 운영 신뢰도가 유지되는 기업
Streaming ad platform 중 ad users·ad revenue·ad ARPU·privacy risk가 통과된 기업

Watch-to-Green:
컨택센터 AI
AI application
ad-tech platform
게임 IP portfolio
플랫폼 기업
생체인식·딥페이크 보안
키오스크·서비스 자동화

Watch/Red:
전통 광고대행
단일 IP 게임사
UGC game platform safety risk
NFT·메타버스
AI 기능만 있는 테마
유저 수만 많은 플랫폼
광고 회복만 있는 media cycle

Hard 4C:
보안 대형 장애
개인정보·광고품질 소송
청소년 안전성 규제
단일 신작 출시 지연
창업자·M&A 법적 리스크
광고 경기 급랭
AI disruption으로 기존 광고·교육·콘텐츠 model 훼손
scam ads / high-risk ad revenue
```

**R8 Loop 3 점수정규화의 핵심 문장:**

> 플랫폼·콘텐츠·SW·보안은 “유저 수”, “AI 기능”, “신작 기대”, “보안 수요”가 아니라 **반복매출, ARR/ARPU, bookings, billings, OPM, FCF, 낮은 churn, 운영 신뢰도, 법적 안정성, 광고 매출 품질**로 봐야 한다.
> 실적이 좋아도 valuation이 포화되면 4B이고, 보안사고·개인정보·청소년 안전·광고품질·출시 지연·창업자 법적 리스크가 나오면 즉시 RedTeam이다.
